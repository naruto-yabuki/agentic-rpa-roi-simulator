## 0. 最優先の回答：「人数」「件数/日」の2問だけで試算できるか

### 0-1. 結論

**2問だけでは、資料の計算式に基づく試算はできない。ただし「業界別デフォルト値」を組み込めば、顧客への質問は実質2問のままで試算可能になる。**

資料の計算式は次の通り：

```
月間削減余地 = 平均人件費（円/時） × 自動化可能工数（時間/月）
```

※資料では平均人件費を2,500円/時と明記しているが、本ツールでは「平均年収600万円」をデフォルトとし、年間所定労働時間2,000時間で除して時給換算（600万円 ÷ 2,000時間 = **3,000円/時**）する方式を採用する（5-2参照）。資料値の再現が必要な場合は設定ページで平均年収を500万円（=2,500円/時）に変更できる。

この式の唯一の変数は「自動化可能工数」であり、これを求めるには **対象業務に投入されている総工数（時間/月）** が必要。ところが、

- **人数** だけでは工数にならない（担当者がその業務に1日何時間かけているかが不明。専従とは限らない）
- **件数/日** だけでも工数にならない（**1件あたりの処理時間** が不明）

つまり、2問のどちらのルートでも「工数」への変換係数が1つ欠けている。

### 0-2. 入力項目の整理（必須／推奨／デフォルト代替）

**A. 計算に最低限必要な入力項目（これがないと計算不能）**

| # | 項目 | 入手方法 |
|---|---|---|
| A1 | 業界 | **顧客に質問しない。** 営業が商談前・商談中に画面上で選択する。業界別デフォルト値（1件あたり処理時間・自動化率）を引き当てるためのキー |
| A2 | 担当人数 | 質問①（顧客ヒアリング） |
| A3 | 処理件数/日 | 質問②（顧客ヒアリング） |
| A4 | 1件あたり処理時間 | **業界別デフォルト値で自動補完**（顧客に聞かなくても計算は成立する） |

- A4は業界デフォルトで代替できるため、**顧客への口頭質問は①②の2問のままで試算が成立する設計にできる。**
- ただしA4の精度が試算額を直接左右するため、後述の「確認型質問」で可能な限りその場で上書きすることを強く推奨する。

**B. あると精度・説得力が上がる入力項目（質問3問目以降の候補）**

| # | 項目 | デフォルト値 | 質問する価値 |
|---|---|---|---|
| B1 | 1件あたり処理時間（A4の上書き） | 業界別デフォルト | **最優先。** 「同業他社では1件あたり◯分程度ですが、御社もそのくらいですか？」という**確認型の聞き方**なら顧客の負担が小さく、かつ「顧客自身が申告した数字」になるため試算の納得感が跳ね上がる |
| B2 | 平均人件費 | 平均年収600万円 ÷ 年間所定労働時間2,000時間 = 3,000円/時 | 一次商談で給与の話は聞きにくいため顧客には質問しない。営業が商談前に**設定ページ**で顧客の給与水準に合わせて平均年収を調整しておく運用とする |
| B3 | 月間稼働日数 | 20日 | ほぼ全社共通。質問不要（設定ページで調整可） |
| B4 | 自動化率 | 業界別デフォルト（75〜85%） | 顧客には判断不能な数値。ニューロスフィア側の実績値なので質問不要（設定ページで調整可） |
| B5 | 初期投資・月額運用費 | 300万円／50万円 | 自社の価格。質問不要（設定ページで調整可） |

※B2〜B5はすべて顧客に質問せず、**設定ページ（顧客に見せない画面）で営業が事前調整**する設計とする（6章参照）。

**C. 人数（A2）の本当の使い道**

人数は工数計算の主変数ではなく、以下3つの用途で使う。これが本ツールの設計上の要点：

1. **妥当性チェック（キャップ）**：件数×処理時間から算出した月間工数が「人数 × 8時間 × 稼働日数」の物理上限を超えたら警告し、上限値で頭打ちにする（過大試算による信頼失墜を防ぐ）
2. **専従率の提示**：「この業務が担当者の稼働の◯%を占めています」という気づきを与える
3. **「◯人分の工数に相当」表示**：削減効果を人数換算して直感的に伝える

### 0-3. 推奨する質問設計（営業の口頭ヒアリング台本）

| 順 | 質問 | 種別 |
|---|---|---|
| ― | （営業が画面で業界を選択。質問ではない） | 操作 |
| ① | 「その事務作業は、何名の方が担当されていますか？」 | 必須 |
| ② | 「1日あたり、合計で何件くらい処理されていますか？」 | 必須 |
| ③ | 「同業の企業様だと1件あたり◯分くらいかかるケースが多いのですが、御社も同じくらいの感覚ですか？」 | 推奨（確認型・答えが返ればスライダーで上書き、返らなければデフォルトのまま） |

**質問は最大3問、うち3問目は「Yes/Noで答えられる確認」**に落とすことで、一次商談でも自然に使える。

## 1. デモ/ツールの目的

- 一次商談の場で、顧客の実数値（担当人数・処理件数）を2〜3問ヒアリングするだけで、**Agentic RPA導入によるコスト削減額・ROI回収期間をその場で算出し、画面で見せる**Webツールを提供する。
- 提案資料の静的な業界別試算表（「卸売なら約140万円/月」）を、**「御社の場合は◯◯万円/月」という顧客固有の数字**に変換することが目的。
- 商談後にはその場の試算結果をPDF（A4・1枚）として持ち帰り資料化できる。

## 2. 伝えたいメッセージ・ゴール

### 伝えたいメッセージ
- 「御社のその事務作業には、月◯◯万円分の削減余地が眠っている」
- 「初期投資300万円は約◯ヶ月で回収できる」
- 「人は承認と例外対応に集中し、転記・照合はAIが代行する」（金額の根拠が"自動化可能工数"であることをビジュアルで示す）

### 一次商談で営業がどう有利になるか
1. **その場で顧客固有の数字が出る**：一般論の資料説明から「自社ごと」の議論に転換でき、二次商談（業務棚卸しの提案）への誘導材料になる
2. **顧客の申告値を使う**：人数・件数は顧客自身が答えた数字なので、試算結果への反論が起きにくい
3. **持ち帰り資料が即時生成される**：顧客の担当者が社内（決裁者）へ展開しやすいPDFを渡せる
4. **ヒアリング品質の平準化**：営業の経験値に依らず、誰でも同じ精度の試算・トークができる

### ゴール（デモの成功条件）
- 業界選択→2〜3問入力→結果表示まで**60秒以内**で完了できること
- 結果画面を見た顧客が「その前提はどこから来たのか」と聞いたときに、前提条件がすべて画面上で確認・調整できること

## 5. 計算ロジック

すべてクライアントサイドで同期計算。入力変更のたびに即時再計算する。

### 5-1. 記号定義

| 記号 | 内容 | 由来 |
|---|---|---|
| N | 担当人数（人） | IN-2 |
| Q | 処理件数（件/日） | IN-3 |
| T | 1件あたり処理時間（分/件） | IN-4（初期値はSET-5） |
| A | 平均年収（円/年） | SET-1、既定6,000,000 |
| L | 年間所定労働時間（時間/年） | SET-2、既定2,000（月160〜170時間×12ヶ月に相当する一般的な所定労働時間として採用） |
| W | 時給換算人件費（円/時） | **導出値**：W = A ÷ L（既定 6,000,000÷2,000 = 3,000円/時） |
| D | 月間稼働日数（日） | SET-3、既定20 |
| h | 1日の所定労働時間（時間/日） | SET-4、既定8 |
| r | 自動化率（0〜1） | SET-6（メイン画面では業界に応じ自動適用） |
| C₀ | 初期投資（円） | SET-8、既定3,000,000 |
| Cm | 月額運用費（円/月） | SET-9、既定500,000 |

### 5-2. 計算式（この順序で実装）

```
(0) 時給換算（円/時）              W = A ÷ L
      ※既定：600万円 ÷ 2,000時間 = 3,000円/時。
        資料の「2,500円/時」は年収500万円相当（500万÷2,000h）。資料値を
        再現する場合は設定ページで平均年収を500万円に変更する（5-5参照）
(1) 月間対象工数（時間/月）        H_raw = Q × T ÷ 60 × D
(2) 物理上限（時間/月）            H_cap = N × h × D
(3) 採用工数                       H = min(H_raw, H_cap)
      ※ H_raw > H_cap のとき「入力値では担当者の総稼働を超えるため、
        総稼働時間で頭打ちにして試算しています」と警告バッジを表示
(4) 業務専従率（%）                occupancy = H ÷ H_cap × 100
(5) 自動化可能工数（時間/月）      H_auto = H × r
(6) 月間削減額（円/月）            S = H_auto × W          ← 資料の式そのもの
(7) 純削減効果（円/月）            P = S − Cm
(8) ROI回収期間（月）              P > 0 のとき ROI = C₀ ÷ P（小数1桁表示）
                                   P ≤ 0 のとき「この業務単体では回収不可」表示（5-4参照）
(9) 年間削減額（円/年）            S_year = S × 12
(10) 人数換算（人分）              FTE_saved = H_auto ÷ (h × D)
(11) 累積純削減額（nヶ月目）       Cum(n) = P × n − C₀   （n = 1..24 を算出しグラフ化）
(12) 黒字転換月                    n_be = Cum(n) ≥ 0 となる最小の整数 n（= ceil(ROI)）
```

### 5-3. 丸め・表示規則

- 金額：万円単位。1万円未満を四捨五入して整数万円で表示（例：1,530,000円 → 「約153万円/月」。内部計算は円のまま保持し、丸めは表示時のみ）
- 時給換算値W：内部計算は生値（除算結果そのまま）を用い、表示時のみ1円未満を四捨五入（例：年収650万円÷2,000h=3,250円/時）
- 工数：時間単位、小数1桁（例：510.0時間/月）
- ROI：小数1桁の「約◯.◯ヶ月」＋グラフ上は n_be ヶ月目にブレークイーブンマーカー
- 人数換算：小数1桁（例：3.2人分）
- 専従率：整数%（100%超は発生しない。式(3)でキャップ済み）

### 5-4. 例外・エッジケース

| ケース | 挙動 |
|---|---|
| P ≤ 0（純削減がマイナス） | ROI・累積グラフの代わりに「この業務単体では月額運用費を下回ります。対象業務の追加（請求・照合・更新管理など周辺業務の同時自動化）で削減余地を広げるのが一般的です」というガイダンスカードを表示。**エラー扱いにしない**（営業トークの分岐材料として設計） |
| H_raw > H_cap | 式(3)の通りキャップ＋警告バッジ。件数か処理時間の再確認を促す文言を添える |
| 業界未選択 | 結果パネルはプレースホルダー（「業界を選択すると試算が始まります」）を表示 |
| 入力が範囲外 | 入力UI側でクランプ（エラーメッセージより誤操作復帰を優先） |

### 5-5. 提案資料との整合性に関する注記（営業チームへの申し送り。実装にも影響）

- **人件費単価の差異**：資料は2,500円/時（年収500万円相当）で試算しているが、本ツールのデフォルトは平均年収600万円（=3,000円/時）。そのため同一工数でも本ツールの試算額は資料の業界別試算表より約2割大きく出る。資料と並べて見せる商談では、設定ページで平均年収を500万円にして揃えるか、「御社の給与水準（年収◯◯万円）で計算しています」と口頭補足すること。
- 資料の「ROI回収期間目安」列は、本式の厳密計算と一致しない箇所がある（例：宿泊業 削減80万円/月 → 純30万円/月 → 厳密には300÷30=10ヶ月だが、資料は〜6.0ヶ月と記載）。
- 本ツールは**厳密式（5-2）を唯一の正**とする。資料の目安値をツールに埋め込まないこと（画面上で資料と数字が食い違うと商談で不信を招くため、資料側の目安値は「参考レンジ」として扱い、ツールには表示しない）。
- 資料実例の「Month4 +280万円」は初期投資差引**前**の累積と解釈される。本ツールの累積グラフは**差引後（式11）**で統一し、凡例に「初期投資300万円控除後」と明記する。

## 8. データ要件

### 8-1. マスタデータ（コード内定数＝「工場出荷値」）

- **業界別ベンチマークマスタ**：4-3のテーブルを唯一のマスタとし、`src/data/industryBenchmarks.ts`（TypeScriptの型付き定数）として一元管理。項目：業界ID、表示名、処理時間デフォルト、自動化率デフォルト、件数デフォルト、代表業務チェーン（文字列配列）、定性効果メッセージ（文字列配列）
- **共通既定値マスタ**：平均年収600万円・年間所定労働時間2,000時間・稼働日20日/月・所定労働8時間/日・初期投資300万円・月額運用費50万円を同ファイルに定数定義（マジックナンバー禁止）。時給はコードに直接持たず、常に「年収÷年間所定労働時間」で導出する
- マスタ（工場出荷値）の恒久的な変更は当該TSファイルの編集＋再デプロイで行う運用とする

### 8-2. 設定値の状態管理（設定ページ→メイン画面の反映）

- マスタ定数を初期値として、**設定ページで編集可能な「実効設定値」をReact Context（アプリ全体で共有する単一のSettingsオブジェクト）で保持**する。メイン画面の計算は常にこの実効設定値を参照するため、設定変更は即時反映される
- **DB永続化は不要**（コーディネータ指示による）。メモリ内Contextのみで必須要件は満たす
- あると良い：Contextの内容を`localStorage`に同期し、リロード・ブラウザ再起動でも設定が残るようにする（商談直前のリロード事故対策。実装コストが低いため推奨）
- 「初期値に戻す」＝実効設定値をマスタ定数で上書き（localStorage併用時はストレージもクリア）

### 8-3. データの取り扱い

- 外部API・DB・サーバー送信は**一切なし**。顧客名を含む全入力はブラウザ内のみで完結し、送信・保存しない（この旨を画面フッターに明記すると商談で安心材料になる）

## 9. UI/UXの方針

- **トーン**：B2B・金融レベルの信頼感。提案資料のトーンに合わせ、ネイビー基調＋白背景＋アクセント1色（エメラルド系＝「削減・利益」の色）。装飾的なイラストは使わず、数字とグラフを主役にする
- **タイポグラフィ**：数値は等幅または太字サンセリフ（Noto Sans JP Bold等）で大きく。桁区切りカンマ必須
- **アニメーション**：数値のカウントアップ（300ms程度）とグラフのトランジションのみ。過剰な動きは信頼感を損なうため禁止
- **操作原則**：入力→結果は常に即時反映（「計算する」ボタンを置かない）。Undo可能性より誤操作耐性（クランプ・ステッパー）を優先
- **言語**：日本語のみ。金額は「万円」表記統一
- **信頼性の演出**：計算根拠パネル（6-3-4）・前提条件の開示・免責文言・保守的デフォルト値、の4点で「盛っていない試算」であることを一貫して示す

## 12. サンプルシナリオ（受け入れテスト兼用の計算例）

以下は式(0)〜(12)、既定値（A=600万円・L=2,000時間 → **W=3,000円/時**、D=20日、h=8時間、C₀=300万円、Cm=50万円）による厳密計算値。シナリオ4のみ設定ページで平均年収を変更する（設定反映の確認を兼ねる）。実装後、この数値と一致することを確認すること。

### シナリオ1：卸売（標準ケース）
- 入力：業界=卸売、N=4人、Q=120件/日、T=15分（デフォルト）、r=85%、設定はすべて初期値（W=3,000円/時）
- H_raw = 120×15÷60×20 = **600時間/月**、H_cap = 4×8×20 = 640時間 → キャップ非発動、専従率 **94%**
- H_auto = 600×0.85 = **510時間/月**
- S = 510×3,000 = 1,530,000円 → **約153万円/月**（年間 約1,836万円）
- P = 1,030,000円/月 → ROI = 3,000,000÷1,030,000 = **約2.9ヶ月**、黒字転換 **3ヶ月目**（Cum(2)=−940,000円、Cum(3)=+90,000円）
- 人数換算 = 510÷160 = **3.2人分**

### シナリオ2：物流（標準ケース）
- 入力：業界=物流、N=4人、Q=150件/日、T=12分（デフォルト）、r=83%、設定初期値
- H_raw = 150×12÷60×20 = **600時間/月**、H_cap = 640時間 → 非発動、専従率 **94%**
- H_auto = 600×0.83 = **498時間/月**
- S = 498×3,000 = 1,494,000円 → **約149万円/月**（年間 約1,793万円）
- P = 994,000円/月 → ROI = 3,000,000÷994,000 = **約3.0ヶ月**、黒字転換 **4ヶ月目**（Cum(3)=−18,000円、Cum(4)=+976,000円）
- 人数換算 = **3.1人分**

### シナリオ3：不動産（小規模・回収不可ケース）
- 入力：業界=不動産、N=2人、Q=2件/日、T=240分（デフォルト）、r=80%、設定初期値
- H_raw = 2×240÷60×20 = **160時間/月**、H_cap = 320時間 → 非発動、専従率 **50%**
- H_auto = 128時間/月 → S = 128×3,000 = 384,000円 → **約38万円/月**
- P = **−116,000円/月** → **回収不可ガイダンス表示**（ROI・グラフ非表示、「対象業務の追加提案」カードを表示）
- ※このケースが「エラーにせず営業トークの分岐にする」仕様（5-4）の確認用

### シナリオ4：電力販売（資料の実導入事例の再現。設定ページの反映確認を兼ねる）
- **設定ページ操作：平均年収を500万円に変更**（→ 時給換算 500万÷2,000 = **2,500円/時**。資料の前提値と一致することを設定ページの換算式表示で確認）
- 入力：業界=その他、N=5人、Q=60件/日、T=30分（手動上書き）、r=80%
- H_raw = 60×30÷60×20 = **600時間/月**、H_cap = 800時間 → 非発動、専従率 **75%**
